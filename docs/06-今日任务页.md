# 06. 今日任务页

## 1. 页面定位

今日任务页只负责一件事：帮助用户聚焦今天。

它固定在 Obsidian 右侧侧边栏打开，只作为辅助操作页面使用。

因此它保持：

1. 非全屏化倾向。
2. 轻量列表布局。
3. 快速编辑和快速完成。

## 2. 数据来源

今日任务页从统一 `Task` 数据中运行时展开实例，再筛选：

```text
occurrence.date === 今天
```

## 3. 页面分区

页面默认分为：

1. 未完成任务。
2. 已完成任务。

## 4. 支持的操作

### 4.1 通用操作

1. 勾选完成当前发生。
2. 编辑整条任务。
3. 删除当前发生。

### 4.2 重复任务专属操作

若任务属于重复任务，则额外支持：

1. 删除整个任务。
2. 到本次为止结束重复。
3. 继续保留历史发生记录。

## 5. 关于数据模型

当前实现采用“系列单条存储 + 发生日期集合”的思路，因此：

1. 每日重复任务不会在存储层拆成多条 Task。
2. 每周重复任务也只是一条 Task。
3. 今天页面看到的是运行时展开后的当天发生实例。

## 6. 关于重复任务提前结束

当用户决定今天就结束一个重复任务时：

1. 当前发生以及更早发生会被视为已完成。
2. 未来发生会被直接移除。
3. 不再把未来日期错误地标记为今天完成。

## 7. 与其他页面关系

1. 今日任务页完成任务后，热度图同步变化。
2. 今日任务页修改日期或时间后，周任务图同步变化。
3. 今日任务页修改项目归属后，项目进度页同步变化。

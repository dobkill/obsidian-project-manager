# 05. 项目进度视图

## 1. 页面定位

项目进度页用于按项目集中管理任务。

它属于总览主页面的一部分，必须跟随“任务总览”入口在中间主工作区展示。

它强调：

1. 重复任务不能在表格里拆成多条。
2. 完成进度要一眼可见。
3. 录入速度高于复杂表格技巧。

## 2. 页面结构

每个项目对应一个项目进度页。

页面包含：

1. 项目标签切换。
2. 项目进度条。
3. 项目任务表格。
4. 新增任务入口。

## 3. 表格列意图

当前建议顺序：

```text
任务名称 -> 重复 -> 计划 -> 完成 -> 描述 -> 操作
```

说明：

1. 同一个重复任务只显示一行。
2. “计划”列需要体现时间范围、时间段和总发生次数。
3. “完成”列展示已完成次数 / 总次数。

## 4. 新增任务规则

### 4.1 默认时间

1. 若当天已有排期任务，则默认开始时间取当天最后一个任务的结束时间。
2. 若当天没有排期任务，则使用全局默认开始时间。

### 4.2 重复任务

新增任务支持：

1. 单次任务。
2. 每日重复任务。
3. 每周此时重复任务。

若选择重复任务，必须提供：

1. 重复次数，或
2. 重复结束日期。

## 5. 编辑规则

### 5.1 系列编辑

项目进度页编辑的是整条任务系列。

保存时会整体更新：

1. 标题。
2. 描述。
3. 项目归属。
4. 起始日期。
5. 时间段。
6. 重复规则与发生日期集合。

### 5.2 删除规则

1. 单次任务可直接删除。
2. 重复任务可删除整条任务。
3. 若从实例视角打开编辑器，还可只删除当前发生。

### 5.3 完成规则

1. 项目进度页的进度按“已完成发生次数 / 总发生次数”计算。
2. 提前结束重复时，当前发生及之前发生标记为完成。
3. 当前发生之后的未来发生会被移除。

## 6. 与总览页同步

1. 项目进度页中的 `occurrenceDates` 决定周任务图和今日页会出现哪些实例。
2. 项目进度页中的 `completedOccurrences.completedAt` 会影响热度图与趋势图。
3. 项目进度页只负责系列管理，不再展示重复实例明细。

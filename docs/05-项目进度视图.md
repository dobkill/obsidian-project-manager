# 05. 项目进度视图

## 1. 页面定位

项目进度页用于按项目集中管理任务。

它属于总览主页面的一部分，必须跟随“任务总览”入口在中间主工作区展示。

它强调：

1. 重复任务不能在表格里拆成多条。
2. 完成进度要一眼可见。
3. 录入速度高于复杂表格技巧。

## 2. 页面结构

每个项目对应一个项目进度页。

页面包含：

1. 项目标签切换。
2. 项目进度条。
3. 项目任务表格。
4. 新增任务入口。

## 3. 表格列意图

当前建议顺序：

```text
任务名称 -> 重复 -> 计划 -> 完成 -> 描述 -> 操作
```

说明：

1. 同一个重复任务只显示一行。
2. “计划”列需要体现时间范围、时间段和总发生次数。
3. “完成”列展示已完成进度。
4. 普通任务按“已完成发生次数 / 总发生次数”统计。
5. 组合任务按“已完成子任务数 / 全部子任务数”统计；若带重复，则按所有发生的子任务总量聚合。

## 4. 新增任务规则

### 4.1 默认时间

1. 若当天已有排期任务，则默认开始时间取当天最后一个任务的结束时间。
2. 若当天没有排期任务，则使用全局默认开始时间。

### 4.2 重复任务

新增任务支持：

1. 单次任务。
2. 每日重复任务。
3. 每周此时重复任务。
4. 组合任务。

若选择重复任务，必须提供：

1. 重复次数，或
2. 重复结束日期。

若选择组合任务，还必须提供：

1. 至少一个子任务。
2. 每个子任务标题不能为空。

## 5. 编辑规则

### 5.1 系列编辑

项目进度页编辑的是整条任务系列。

保存时会整体更新：

1. 标题。
2. 描述。
3. 项目归属。
4. 起始日期。
5. 时间段。
6. 重复规则与发生日期集合。

### 5.2 删除规则

1. 单次任务可直接删除。
2. 重复任务可删除整条任务。
3. 项目进度页的“详细编辑”始终视为系列编辑，不提供“删除本次实例”。
4. 只有从周任务图或今日任务页进入、且带明确实例上下文时，才允许删除当前发生。

补充：

1. 系列编辑中的危险操作文案统一使用“删除整个系列”。
2. 避免把系列首个发生误当成“本次”删除。
3. 删除整个项目时，会同时删除该项目下的全部任务系列与发生记录。

### 5.3 完成规则

1. 普通任务的项目进度按“已完成发生次数 / 总发生次数”计算。
2. 组合任务的项目进度按“已完成子任务数 / 总子任务数”计算。
3. 提前结束重复时，当前发生及之前发生标记为完成。
4. 当前发生之后的未来发生会被移除。
5. 项目进度页不再通过弹窗提供“完成状态”总开关，避免整条系列被误批量改完成。

## 6. 与总览页同步

1. 项目进度页中的 `occurrenceDates` 决定周任务图和今日页会出现哪些实例。
2. 项目进度页中的 `occurrenceStates.completedAt` 会影响热度图与趋势图。
3. 项目进度页中的 `occurrenceStates.completedSubtaskIds` 会影响组合任务在今日页和周任务图中的子任务状态。
4. 项目进度页只负责系列管理，不再展示重复实例明细。
5. 删除项目后，关联任务会从项目进度页、周任务图和今日任务页同步消失。

## 7. 周任务图卡片密度

周任务图采用紧凑信息卡片，不再把标签和编辑操作做成独占的大按钮区域。

约定：

1. 任务标题、重复徽标和编辑入口尽量横向合并。
2. 重复徽标使用弱化 chip 样式，减少对标题的抢占。
3. 编辑入口使用图标按钮，默认在 hover 或键盘聚焦时显示；移动端保持可见。
4. 周视图卡片降低 padding、圆角、阴影和左侧色条厚度。
5. 组合任务子任务 chip 使用更小网格，优先保证一屏信息密度。

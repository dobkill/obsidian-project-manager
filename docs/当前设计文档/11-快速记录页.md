# 11. 快速记录页

## 1. 页面定位

快速记录页是右侧侧边栏中的 Quick Capture 面板。

目标不是做完整编辑器，而是用最短路径完成四件事：

1. 写日记。
2. 追加任意 Markdown 笔记。
3. 用多行语法快速建任务。
4. 给思维导图补充正文或评语节点。

## 2. 页面结构

当前页面由四部分组成：

1. 顶部标题和副标题。
2. 四个目标卡片：
   写日记 / 追加笔记 / 创建任务 / 补充导图
3. 与目标相关的路径或节点选择卡片。
   补充导图模式下依次显示：
   目标项目 / 目标节点 / 快捷节点
4. 统一的编辑器卡片：
   工具栏 + textarea + 字数统计 + 提交按钮

## 3. 目标切换规则

### 3.1 写日记

1. 展示当前实际写入路径。
2. 若设置为单文件模式，则写入 `dailyNoteSingleFilePath`。
3. 若设置为按日文件模式，则写入 `dailyNoteFolder/YYYY-MM-DD.md`。

### 3.2 追加笔记

1. 支持最近 Markdown 文件快速选择。
2. 支持手动输入 Vault 内相对路径。
3. 若目标文件不存在，会自动创建父目录与文件。

### 3.3 创建任务

1. 沿用格式化文本解析。
2. 支持 `#项目：新项目名` 自动创建新项目，也支持 `#项目：` 导入未归属任务。
3. 同项目下若任务名重复，则按任务名覆盖旧任务。
4. `- [x]` 默认只完成当天；若是重复任务，可写 `finish:series` 提前结束整个系列。
5. 提交后直接走 `importFormattedTasks`。
6. 写入来源标记为 `dialog`。

### 3.4 补充导图

当前实现把旧逻辑拆成两种明确模式：

1. `创建子节点`
   每一行内容创建一个新的评语节点。
2. `追加到节点正文`
   若当前选中任务节点，则内容追加到任务描述。
   若当前选中评语节点，则内容追加到该评语文本。

节点选择规则：

1. 先选项目。
2. 选中项目后，再展示该项目下的全部任务节点与评语节点。
3. 节点列表会同时包含：
   任务节点本身 + 该任务下的评语树。
4. 页面额外维护“快捷节点”区：
   跨任务共享，只保留最近使用的 6 个节点，并标注所属任务。
5. 提交时会读取最新任务快照，再决定是追加到任务描述还是追加到评语正文，避免节点选择状态过期导致写入失败。

## 4. 编辑器规则

编辑区采用统一 editor card：

1. 工具栏提供粗体、斜体、标题、列表、引用、代码插入。
2. placeholder 会随目标变化。
3. 底部始终显示字数。
4. 提交按钮固定在编辑卡片右下。

## 5. 与思维导图页的关系

快速记录页不负责布局导图节点位置，也不负责修改任务父子关系。

它只负责：

1. 追加任务正文。
2. 追加评语正文。
3. 创建新的评语子节点。

任务父子关系、评语挂载关系和依赖指向，在项目进度页的思维导图检查面板中维护。

# 06. 今日任务页

## 1. 页面定位

今日任务页只负责一件事：帮助用户聚焦今天。

它固定在 Obsidian 右侧侧边栏打开，只作为辅助操作页面使用。

因此它保持：

1. 非全屏化倾向。
2. 以卡片化列表替代旧的扁平按钮排布。
3. 快速编辑和快速完成。

## 2. 数据来源

今日任务页从统一 `Task` 数据中运行时展开实例，再筛选：

```text
occurrence.date === 今天
```

补充：

1. 逾期未完成的单次任务会在 store 层自动顺延到今天，因此也会出现在今日任务页。
2. 顺延后的单次任务会自动占用当天一个不冲突的 1 分钟空档。

## 3. 页面分区

页面默认分为：

1. 顶部标题区：
   标题、日期、`+ 新增任务`
2. 今日进度卡片。
3. 未完成任务区。
4. 已完成任务区。
5. 底部小贴士卡片。

今日进度规则：

1. 有任务时显示环形进度 + 细进度条。
2. 进度按“完成步骤 / 总步骤”统计。
3. 普通任务记为 1 步；组合任务按子任务数累计步骤。
4. 今日无任务时保留 0% 进度卡片文案，但不误导为已有完成记录。

## 4. 支持的操作

### 4.1 通用操作

1. 勾选完成当前发生。
2. 通过更多菜单编辑整条任务。
3. 通过更多菜单删除当前发生。
4. 若为组合任务，则点击子任务小块切换该子任务完成状态。
5. 若从重复任务实例进入编辑，可选择“仅保存本次时间”，只写入当前日期的 `occurrenceOverrides`。

### 4.2 重复任务专属操作

若任务属于重复任务，则额外支持：

1. 删除整个任务。
2. 到本次为止结束重复。
3. 继续保留历史发生记录。
4. 危险操作只在菜单中出现，不默认常驻页面。

### 4.3 今日任务导出

今日任务页新增“导出今日任务”入口。

当前规则改为更聚焦的单一行为：

1. 点击后直接复制“今日未完成任务”。
2. 不再弹出三种导出模式菜单。
3. 导出文本语法仍与快速记录页“创建任务”和项目页“批量导入”共用。
4. 已归属项目会按 `#项目：项目名` 分组导出。
5. 未归属任务会导出为 `#项目：未归属项目` 分组。
6. 由于只导出未完成任务，复制内容不会主动写入 `finish:today` 或 `finish:series`。

## 5. 关于数据模型

当前实现采用“系列单条存储 + 发生日期集合”的思路，因此：

1. 每日重复任务不会在存储层拆成多条 Task。
2. 每周重复任务也只是一条 Task。
3. 组合任务也仍然只是一条 Task，只是额外包含子任务定义和每次发生的子任务完成状态。
4. 今天页面看到的是运行时展开后的当天发生实例。
5. 重复任务某次发生的不同时间来自 `occurrenceOverrides`。

## 6. 视觉规则

1. 未完成和已完成分区都使用卡片容器。
2. 已完成任务使用轻微淡出和划线，不使用高饱和红色操作按钮。
3. checkbox hover 使用紫色柔和描边。
4. 已完成区在空状态下使用图标 + 文案，而不是仅显示“暂无任务”。

## 7. 关于重复任务提前结束

当用户决定今天就结束一个重复任务时：

1. 当前发生以及更早发生会被视为已完成。
2. 未来发生会被直接移除。
3. 不再把未来日期错误地标记为今天完成。

## 8. 与其他页面关系

1. 今日任务页完成任务后，热度图同步变化。
2. 今日任务页修改日期或时间后，周任务图同步变化。
3. 今日任务页修改项目归属后，项目进度页同步变化。
4. 今日任务页修改某次重复任务时间后，项目管理页和周任务图会按覆盖后的时间展示。

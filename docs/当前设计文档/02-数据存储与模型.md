# 02. 数据存储与模型

当前版本采用新数据模型，不兼容旧版结构。软件尚未发布，因此若旧字段与新设计冲突，直接以新模型为准。

## 1. 存储目录

插件数据默认存放在：

```text
project-manager-data/
```

目录结构：

```text
project-manager-data/
├── config.json
├── projects.json
├── project-pages.json
├── note-task-index.json
├── write-history.json
└── tasks/
    ├── 2026-05.json
    └── 2026-06.json
```

说明：

1. `tasks/YYYY-MM.json` 保存任务真实数据。
2. `project-pages.json` 保存项目页排序和列配置。
3. `note-task-index.json` 保存全局笔记任务块扫描缓存。
4. `write-history.json` 保存批量导入、自动排程、笔记同步等批量写入记录。

## 2. 配置模型

```ts
type PluginConfig = {
  version: string;
  dataFolder: string;
  overviewTab1Name: string;
  overviewTab2Name: string;
  dialogTabName: string;
  weekStartsOn: "monday";
  timeSlotMinutes: number;
  heatmapRange: "12months";
  showCompletedTasks: boolean;
  defaultTaskDurationMinutes: number;
  defaultTaskStartTime: string;
  dailyNoteFolder: string;
  dailyNoteDateFormat: string;
  dailyNoteMode: "per-day" | "single-file";
  dailyNoteSingleFilePath: string;
  taskNoteRecentLimit: number;
  defaultDialogTarget: "daily-note" | "task-note" | "quick-task" | "mindmap";
};
```

新增配置：

1. `dialogTabName` 控制右侧快速记录页名称。
2. `dailyNoteFolder` 控制快速记录写入日记的位置。
3. `dailyNoteMode` 与 `dailyNoteSingleFilePath` 控制日记写入策略。
4. `taskNoteRecentLimit` 控制“追加笔记”展示的最近文件数量。
5. `defaultDialogTarget` 控制快速记录默认保存目标。

## 3. 任务模型

任务是唯一真实数据源。表格、看板、甘特图、思维导图都通过同一个 `taskId` 读取和修改任务。

```ts
type Task = {
  id: string;
  kind: "simple" | "composite";
  title: string;
  description?: string;
  projectId?: string;
  status: "todo" | "doing" | "blocked" | "done";
  priority?: "low" | "medium" | "high" | "urgent";
  tags: string[];

  date: string;
  startTime?: string;
  endTime?: string;
  recurrence: "once" | "daily" | "weekly" | "custom";
  recurrenceCount?: number | null;
  recurrenceUntil?: string | null;

  subtasks: Array<{ id: string; title: string; order: number }>;
  occurrenceDates: string[];
  occurrenceStates: TaskOccurrenceState[];
  occurrenceOverrides: TaskOccurrenceOverride[];

  viewState: TaskViewState;
  sourceLinks: TaskSourceLink[];
  notes: TaskNote[];

  createdAt: string;
  updatedAt: string;
  revision: number;
};
```

## 4. 单次发生覆盖

重复任务默认使用系列级 `date/startTime/endTime`。若某一次发生需要不同时间，写入：

```ts
type TaskOccurrenceOverride = {
  date: string;
  startTime?: string;
  endTime?: string;
  title?: string;
  skipped?: boolean;
  reason?: string;
};
```

运行时展开实例时，优先使用 `occurrenceOverrides` 中同日期的覆盖字段。

## 5. 四视图状态

```ts
type TaskViewState = {
  board: {
    columnId: "todo" | "doing" | "blocked" | "done";
    order: number;
  };
  gantt: {
    rowOrder: number;
    dependencyIds: string[];
    locked: boolean;
    milestone: boolean;
  };
  mindmap: {
    parentTaskId?: string | null;
    childOrder: number;
    expanded: boolean;
    x?: number;
    y?: number;
  };
};
```

规则：

1. 表格视图只编辑任务主字段，不保存独立任务副本。
2. 看板视图读写 `status` 与 `viewState.board`。
3. 甘特图读写日期、时间与 `viewState.gantt`。
4. 思维导图读写 `viewState.mindmap`。

## 6. 来源与笔记

```ts
type TaskSourceLink = {
  id: string;
  type: "note" | "daily-note" | "dialog" | "manual";
  path?: string;
  blockId?: string;
  line?: number;
  hash?: string;
  syncMode: "import-only" | "linked" | "mirror";
  lastSyncedAt?: string;
  missing?: boolean;
};
```

用途：

1. 批量导入和快速记录可以记录来源。
2. 全局笔记任务块通过 `path + hash` 与任务建立关联。
3. 来源文件删除或任务块消失时，先标记来源失效，不直接删除任务。

## 7. 运行时实例

页面层会把 `Task` 展开成 `TaskOccurrence`。

展开规则：

1. 遍历 `occurrenceDates`。
2. 跳过 `occurrenceOverrides.skipped === true` 的日期。
3. 使用覆盖后的标题、开始时间、结束时间。
4. 读取 `occurrenceStates` 计算完成状态。
5. 普通任务按 1 步计算，组合任务按子任务数量计算。

## 8. 写入规则

1. 任务新增、编辑、删除、完成状态变化会写入对应 `tasks/YYYY-MM.json`。
2. 项目新增、编辑、删除会写入 `projects.json` 与 `project-pages.json`。
3. 全局笔记同步会更新 `note-task-index.json`。
4. 批量导入、自动排程、笔记同步会追加 `write-history.json`。
5. 所有写入仍走串行写入队列，写入后回读磁盘并刷新视图。

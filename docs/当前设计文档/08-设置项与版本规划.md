# 08. 设置项与版本规划

## 1. 当前设置项

当前设置页支持以下内容：

1. 数据目录路径。
2. 任务总览标签名称。
3. 项目进度标签名称。
4. 时间粒度。
5. 默认任务时长。
6. 默认开始时间。
7. 是否显示已完成任务。
8. 快速记录页名称。
9. 日记文件夹。
10. 快速记录默认目标。

设置页顶部还会提供一份“批量导入数据范例”，并补充说明：

1. 范例改为可编辑文本框，但不会保存；每次重新打开设置页都会恢复默认内容。
2. 设置页会实时展示该范例在表格、看板、甘特图和完成统计上的摘要结果。
3. `#项目：新项目名` 可用于演示自动建项目；`#项目：` 可用于演示未归属任务。
4. `- [x]` 可直接导入已完成任务；`finish:series` 可演示重复任务提前结束系列。
5. `status / priority / tags / repeat / 子任务` 会分别影响表格、看板、甘特图与完成度展示。
6. 思维导图中的评语节点、父子关系与依赖关系，导入后在项目进度页或快速记录页继续补充。

## 2. 当前版本完成的重构

当前版本已经完成以下方向：

1. 任务总览页主内容区打开。
2. 热度图日期标识增强。
3. 周任务图列头重做。
4. 新增任务默认时间智能推算。
5. 项目进度页“完成”列前置。
6. 重复任务支持。
7. 系列级删除与完成支持。
8. 数据目录切换支持加载目标目录已有有效数据，或在目标目录无效/为空时创建新数据文件。
9. 数据变更会立即进入写入队列，成功落盘后再刷新视图。
10. 任务模型扩展为状态、优先级、标签、单次发生覆盖、四视图状态、来源链接和任务笔记。
11. 项目管理页新增表格、看板、甘特图、思维导图四视图。
12. 支持格式化文本批量导入任务。
13. 支持右侧快速记录页。
14. 支持全局 Markdown 项目任务块后台扫描。
15. 支持今日自动排程并记录写入历史。

## 4. 当前明确放弃的目标

在当前代码基础上，已不再追求旧格式兼容。

这意味着：

1. 旧任务 JSON 结构不是设计约束。
2. 后续文档以新模型为准。

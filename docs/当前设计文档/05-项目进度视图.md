# 05. 项目进度页

## 1. 页面定位

项目管理页用于按项目集中管理任务。它属于总览主页面的一部分，跟随“任务总览”入口在中间主工作区展示。

当前项目进度页包含四种视图：

1. 表格视图。
2. 看板视图。
3. 甘特图视图。
4. 思维导图视图。

四种视图都读写同一条 `Task`，不能保存独立副本。

## 2. 顶部摘要

当前项目页顶部新增整体进度摘要卡。

摘要卡由两部分组成：

1. 左侧：项目名、说明、整体进度条、完成率。
2. 右侧：项目任务、总次数、已完成、完成率四个指标。

## 3. 表格视图

表格视图负责完整字段编辑和批量入口。

当前列：

```text
任务名称 -> 状态 -> 优先级 -> 标签 -> 重复 -> 计划 -> 完成 -> 描述 -> 操作
```

支持：

1. 查看任务主字段。
2. 以 badge 展示状态、优先级、标签和重复类型。
3. 在完成列中展示“次数 / 比例 + 细进度条”。
4. 通过操作菜单进入详细编辑、状态移动和删除。
5. 底部分页与总数统计。
6. 批量导入格式化文本。
7. 触发今日自动排程。

## 4. 看板视图

看板列：

```text
待办 -> 进行中 -> 阻塞 -> 已完成
```

数据规则：

1. 列由 `status` 和 `viewState.board.columnId` 决定。
2. 卡片支持原生拖拽跨列，落下后会同步更新状态与列归属。
3. 看板卡片展示标题、优先级、标签、排期、完成度和进度条。
4. 阻塞列可显示依赖提醒 badge。
5. 每列顶部支持快速新建任务，底部提供“+ 新建任务”入口。
6. 看板底部显示总任务、进行中、已完成、完成率、逾期统计。
7. 看板状态变化会同步到表格、甘特图和思维导图。

## 5. 甘特图视图

甘特图改为按任务系列展示图形化日期跨度，不再只是纯文本时间条。

当前实现：

1. 结构重构为 `header + left table + right timeline + minimap + legend`。
2. 左侧固定为任务信息表格，列为“任务 / 状态 / 优先级 / 计划”。
3. 右侧时间轴与左侧共用统一 `rowHeight`，任务行与条带严格水平对齐。
4. 时间轴支持 `日 / 周 / 月` 三种粒度，默认按任务总跨度自动选择，更偏向周/月聚合。
5. 支持缩放、适应、重置、今天定位、横向滚动、Shift+滚轮横向滚动、Ctrl/Cmd+滚轮缩放。
6. 长周期任务不再用“每天一个大列”直接撑满，而是通过周/月聚合和缩放自适应展示。
7. 条带颜色继承任务状态；支持锁定、里程碑、依赖数量提示和 today 线。
8. 底部提供 mini map，用于快速感知当前视口在整段时间轴中的位置。

补充：

1. 时间跨度默认取 `date -> defaultCompletionDate(task)`，范围再按粒度自动前后扩展。
2. 周视图主刻度显示周区间，月视图主刻度显示月份，日视图只在短跨度项目中默认启用。
3. 重复任务某一次发生的不同时间仍由 `occurrenceOverrides` 表示。
4. 单次发生时间覆盖从周任务图或今日任务页进入编辑。

## 6. 思维导图视图

思维导图以项目为根节点，以任务为节点。

数据规则：

1. 父子关系写入 `viewState.mindmap.parentTaskId`。
2. 排序写入 `viewState.mindmap.childOrder`。
3. 评语节点数据写入 `mindmapComments`。
4. 快速记录页的“补充导图”只追加正文或创建评语节点，不负责创建任务。
5. 思维导图父子关系表示任务拆解，不等同于甘特图依赖关系。

当前交互：

1. 导图渲染在独立 viewport 内，不再依赖超大滚动画布。
2. viewport 内部包含统一 transform layer，节点和 SVG 连线始终一起缩放和平移。
3. 顶部工具栏支持：
   缩小 / 当前缩放比例 / 放大 / 适应 / 重置 / 新增根任务
4. 支持 `Ctrl/Cmd + 鼠标滚轮` 以鼠标位置为中心缩放。
5. 支持拖拽空白画布平移，节点本身拖拽不会触发画布平移。
6. 打开导图、切换项目、切换到导图视图、viewport 真实尺寸变化后，会自动执行 `fitView()`。
7. 节点拖拽保存后不会再次强制跳回全局总览视角，保留用户当前缩放和平移状态。
8. 任务节点除标题外，还会显示任务正文摘要，便于直接确认“追加到节点正文”的结果。
9. 右侧检查面板改为独立 grid 列，不再覆盖导图内容；窄屏时自动下移。
10. 节点支持拖拽改位置，拖拽结果仍会写回任务或评语的 `x/y`。
11. 点击节点后，右侧检查面板提供真实编辑入口。
12. 任务节点支持：
   编辑任务 / 新增子任务 / 新增评语 / 转为组合任务 / 修改上级任务 / 修改依赖指向
13. 评语节点支持：
   改写 / 新增子评语 / 删除 / 修改挂载父节点
14. 评语节点不再展示“进行中”等无关信息。

## 7. 批量导入

项目管理页支持格式化文本批量导入。

示例：

```text
- [ ] 开发解析器 @2026-05-18 09:00-10:30 #parser !high status:doing
  - 解析标题
  - 解析日期
```

导入流程：

1. 弹窗中输入格式化文本。
2. 实时预览可导入任务和问题行。
3. 预览区会明确展示新增、覆盖、完成今日、提前结束系列、新建项目等结果。
4. 同一项目下若任务名重复，则按任务名覆盖旧任务，而不是重复新增。
5. `- [x]` 默认表示“完成当天这次发生”；若是重复任务，可额外写 `finish:series` 提前结束整个系列。
6. 当前项目页打开的批量导入弹窗默认锁定到当前项目；未写项目头的文本会按当前项目处理。
7. 若导入任务时间与现有任务冲突，会自动改为同日 1 分钟空档占位，不改变重复次数。
8. 确认后按预览结果逐条新增或覆盖。
9. 写入 `write-history.json` 记录本次导入。

## 8. 自动排程

当前实现提供“今日自动排程”入口。

规则：

1. 默认只处理当天已排期、未完成、未锁定任务。
2. 自动向后顺延冲突任务。
3. 若任务被甘特图锁定，则不会移动。
4. 排程结果写入 `occurrenceOverrides` 或系列时间字段。
5. 排程记录写入 `write-history.json`。

## 8.1 单次任务顺延

当前版本增加了单次任务的逾期顺延规则：

1. 若 `recurrence === "once"` 且任务仍未完成，日期早于今天时，会自动顺延到今天。
2. 顺延时会自动寻找当天不冲突的 1 分钟空档占位。
3. 该规则会在数据加载后统一规范化，因此表格、看板、甘特图与今日任务页读到的是同一结果。

## 9. 完成度

完成度统一按步骤计算：

1. 普通任务每次发生记 1 步。
2. 组合任务每个子任务记 1 步。
3. 项目进度、今日进度、看板卡片、甘特条和思维导图节点使用同一套算法。

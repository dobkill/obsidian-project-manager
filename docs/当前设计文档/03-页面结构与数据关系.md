# 03. 页面结构与数据关系

## 1. 页面关系

当前产品包含四个主要工作界面：

```text
任务总览页
├── 热度图
├── 周任务图
└── 最近 30 天趋势图

项目管理页
├── 表格视图
├── 看板视图
├── 甘特图视图
└── 思维导图视图

今日任务页
└── 今日任务列表

快速记录页
├── 写入每日日记
├── 追加到任务笔记
├── 快速创建任务
└── 扩充思维导图
```

入口归属规则：

1. “任务总览页 / 项目管理页”固定在 Obsidian 中间主工作区打开。
2. “今日任务页”固定在 Obsidian 右侧侧边栏打开。
3. “快速记录页”固定在 Obsidian 右侧侧边栏打开。

## 2. 统一数据源

所有视图共享同一套 `Task` 数据。

区别在于：

1. 表格视图编辑任务主字段。
2. 看板视图编辑 `status` 与 `viewState.board`，并支持拖拽跨列。
3. 甘特图视图读取日期跨度、依赖、锁定和里程碑。
4. 思维导图视图读取 `viewState.mindmap` 与 `mindmapComments` 形成任务树和评语树。
5. 今日任务页读取运行时展开后的 `TaskOccurrence`。
6. 总览页读取 `TaskOccurrence` 做时间统计。

## 3. 四类数据联动

同一条任务包含四类数据：

```text
Task
├── 任务主数据：标题、描述、项目、状态、优先级、标签、子任务、完成状态
├── 看板数据：所在列、列内排序
├── 甘特图数据：时间、依赖、锁定、里程碑
└── 思维导图数据：父任务、子任务顺序、展开状态、节点位置
```

严格规则：

1. 四类数据都通过同一个 `taskId` 关联。
2. 任意视图不能创建只属于自己的隐藏任务。
3. 任意视图修改任务后，统一写回任务仓库。
4. 写入完成后触发所有相关视图刷新。

## 4. 典型同步行为

1. 表格修改标题后，看板卡片、甘特条、思维导图节点同步变化。
2. 看板拖拽移动状态后，表格状态列、甘特条颜色与思维导图任务节点同步变化。
3. 甘特图锁定任务后，自动排程不会移动该任务。
4. 今日任务页完成子任务后，项目进度、看板完成度、甘特进度和思维导图任务节点同步变化。
5. 快速记录页向思维导图扩充内容时，只会追加任务描述或创建评语节点，不会偷偷新建任务。

## 5. 全局笔记任务

全局笔记任务不是独立数据源。

同步流程：

```text
Markdown 任务块
-> 格式化文本解析
-> 写入 Task
-> 建立 sourceLinks
-> 写入 note-task-index.json
-> 刷新项目页、今日页和总览页
```

当前识别 `<!-- pm:start -->` 与 `<!-- pm:end -->` 包裹的任务块。

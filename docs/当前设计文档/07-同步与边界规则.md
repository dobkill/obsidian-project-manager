# 07. 同步与边界规则

## 1. 基本原则

所有页面共享同一套任务数据，因此不存在“某个页面保存一份独立任务”的概念。

数据变更采用“先写文件、后刷新视图”的顺序。每次新增、修改、删除或完成状态变更都会进入串行写入队列，目标 JSON 写入成功后才触发页面重新渲染。

## 2. 新增任务

### 2.1 单次任务

创建后只生成一个发生日期。

### 2.2 重复任务

创建后会在同一条 Task 内记录多个发生日期。

生成上限受以下条件约束：

1. 重复次数。
2. 重复结束日期。

## 3. 修改任务

### 3.1 修改单次任务

直接更新当前任务。

### 3.2 修改重复任务

当前版本统一按整条任务系列修改：

1. 重算 `occurrenceDates`。
2. 保留仍然存在的完成记录。
3. 移除新规则下已经不存在的完成记录。

### 3.3 修改单次发生时间

从今日任务页或周任务图的实例上下文进入时，可以选择“仅保存本次时间”。

规则：

1. 不修改系列级 `startTime/endTime`。
2. 写入对应日期的 `occurrenceOverrides`。
3. 时间冲突校验以最终展开后的实例为准。
4. 项目管理页、今日任务页、周任务图都会读取覆盖后的时间。

## 4. 删除任务

### 4.1 删除单次发生

只删除 `occurrenceDates` 中对应的一天。

### 4.2 删除整个任务

直接删除整条 Task。

## 5. 完成任务

### 5.1 完成本次发生

只修改当前发生的：

1. 完成状态。
2. `completedAt`

### 5.2 到本次为止结束重复

会把当前发生及之前发生保留下来并标记完成，同时移除未来发生。

## 6. 热度图统计边界

热度图只统计：

```text
实例 completed === true
并且 completedAt 存在
```

任务计划日期不决定热度图颜色。

## 7. 时间冲突边界

若任务同时具备：

1. `startTime`
2. `endTime`

则在同一天内必须通过时间冲突校验。

不允许：

1. 结束时间早于开始时间。
2. 与同一天已有排期任务重叠。
3. 时间不对齐设定的时间粒度。

## 8. 数据目录切换边界

设置页修改数据目录后：

1. 插件会先检查目标目录是否为 Vault 内相对路径。
2. 如果目标目录已有插件数据，且 `config.json`、`projects.json`、`project-pages.json`、`note-task-index.json`、`write-history.json` 与 `tasks/*.json` 的现有文件都能按当前结构解析，则加载目标目录数据。
3. 如果目标目录不存在、为空，或现有插件数据格式错误，则用当前内存数据在目标目录写入新的插件数据文件。
4. 如果目标目录包含非插件文件，仍会拒绝使用，避免覆盖用户自己的文件。

## 9. 写入失败边界

1. 数据文件读取失败时，插件进入只读保护，不再自动覆盖异常文件。
2. 单次写入失败会让当前操作失败并显示错误；下一次保存仍会继续尝试写入，不会因为队列前一次失败而被永久跳过。
3. 插件卸载时会等待已进入队列的写入完成，并在非只读状态下额外写入一次当前数据快照。

## 10. 启动与入口刷新边界

1. 插件启动时会先按当前设置的数据目录加载一次文件，并在 workspace ready 后再补一次回读。
2. 点击“项目总览”和“今日任务”入口时，不会阻塞当前页面打开；会先显示缓存，再在回读完成后刷新页面。
3. 启动期如果 Vault 缓存还没索引到目标目录或 `tasks/*.json`，插件会回退到磁盘适配器直接读取，避免误判为空目录。
4. 若数据目录在启动期已经存在，即使 Obsidian 内部再次触发目录创建，也应直接忽略“目录已存在”场景，不再把它当作初始化失败。

## 11. 全局笔记任务同步

当前版本支持扫描 Markdown 中的项目任务块：

```md
<!-- pm:start -->
- [ ] 写解析器 @2026-05-18 09:00-10:30 #plugin
<!-- pm:end -->
```

规则：

1. Obsidian workspace ready 后后台扫描，不阻塞页面打开。
2. Markdown 文件修改后会延迟触发单文件同步。
3. 文件 hash 未变化时跳过解析。
4. 解析结果写入插件任务模型，并用 `sourceLinks` 记录来源。
5. 索引缓存写入 `note-task-index.json`。
6. 来源消失时先标记为 missing，不直接删除任务。

## 12. 自动排程边界

自动排程目前作为显式操作提供。

规则：

1. 默认处理当天已排期、未完成、未锁定任务。
2. 当前实现向后顺延冲突任务。
3. 甘特图锁定的任务不会被移动。
4. 移动记录写入 `write-history.json`。

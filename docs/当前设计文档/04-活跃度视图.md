# 04. 任务总览页

## 1. 页面定位

任务总览页是插件主入口。

它必须：

1. 在主内容区打开。
2. 以全屏方式展示。
3. 固定包含热度图、周任务图和最近 30 天趋势图。

## 2. 热度图

### 2.1 展示内容

热度图展示最近 12 个月的完成活跃度。

### 2.2 交互要求

1. 显示月份标识。
2. 显示星期标识。
3. 鼠标悬停显示完整日期与完成数。
4. 点击某一天打开任务明细弹窗。

### 2.3 统计规则

1. 只按实例的 `completedAt` 日期统计完成量。
2. 未完成实例不会提升热度值。
3. 同一天完成的实例越多，颜色越深。

## 3. 周任务图

### 3.1 展示要求

1. 每列标题采用左右结构，左侧显示 `周X`，右侧显示 `YYYY-MM-DD`。
2. “新增任务”入口收纳在列头右侧的小按钮中，不再占满整列宽度。
3. 今天列高亮。
4. 已经过去的日期灰显。
5. 周末轻微区分底色。
6. 每日卡片必须有最小高度，避免空周塌陷。
7. 空日期列使用居中的空态卡片提示“暂无任务”，避免整周空态时出现贯穿所有列的横向视觉线。
8. 不再额外显示“今天”标签，今天状态只通过高亮边框表达。

### 3.2 新增任务规则

1. 若当天已有排期任务，则默认开始时间为当天最后一个任务的结束时间。
2. 默认结束时间 = 默认开始时间 + 默认时长。
3. 若当天没有任务，则默认开始时间使用设置中的 `defaultTaskStartTime`。

### 3.3 周任务图中的重复任务

1. 单次任务。
2. 每日重复。
3. 每周此时重复。
4. 组合任务仍可叠加上述重复规则。

### 3.4 周任务图中的实例操作

1. 勾选完成仅影响当前这次发生。
2. 编辑时打开整条任务系列。
3. 可删除当前发生，或删除整个任务。
4. 可执行“到本次为止结束重复”。
5. 若为组合任务，则当前实例中的子任务可以分别点击完成。

### 3.5 组合任务展示

1. 组合任务使用一个大卡片包裹多个子任务小块。
2. 子任务小块的完成状态独立切换。
3. 卡片顶部显示当前实例的子任务完成数，例如 `1/3 子任务`。
4. 只有当前实例全部子任务完成后，这次发生才视为完成。

## 4. 最近 30 天趋势图

趋势图用于展示：

1. 每日任务总数。
2. 每日完成数量。

两条线都基于运行时展开后的任务实例统计。

当前图表还包含：

1. 图例。
2. 左侧纵轴刻度。
3. 横向辅助线。
4. 悬停点位时可查看当天任务数与完成数。

补充规则：

1. 纵轴刻度始终为整数，并根据最近 30 天最大任务数自动计算。
2. 左上角“任务总数”和“已完成”文本颜色与对应折线保持一致。

## 5. 总览与项目页关系

任务总览页用于看时间与活跃度，不负责完整项目管理。

项目级管理任务应切换到项目进度页处理。

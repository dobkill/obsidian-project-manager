# 03. 页面结构与数据关系

## 1. 页面关系

当前产品包含三个主要工作界面：

```text
任务总览页
├── 热度图
├── 周任务图
└── 最近 30 天趋势图

项目进度页
└── 项目级任务表格

今日任务页
└── 今日任务列表
```

入口归属规则：

1. “任务总览页”入口固定在 Obsidian 中间主工作区打开。
2. “今日任务页”入口固定在 Obsidian 右侧侧边栏打开。
3. 周任务图、热度图、趋势图、项目进度表都归属总览主页面。

## 2. 统一数据源

三个页面共享同一套 `Task` 数据。

区别在于：

1. 项目进度页直接读取系列级 Task。
2. 今日任务页读取运行时展开后的 TaskOccurrence。
3. 任务总览页同时读取 TaskOccurrence 做时间统计与可视化。

## 3. 数据关系

```text
Project
  └── Task.projectId

Task
  ├── occurrenceDates[]
  ├── subtasks[]
  └── occurrenceStates[]
```

## 4. 页面对 Task 的使用方式

### 4.1 任务总览页

1. 热度图读取 `completedAt` 的日期。
2. 周任务图读取实例的 `date`、`startTime`、`endTime`。
3. 趋势图统计最近 30 天的每日任务总数与每日完成数，并使用自动计算的整数纵轴刻度。

### 4.2 项目进度页

1. 按 `projectId` 筛选系列任务。
2. 同一个重复任务只显示一行。
3. 一行内集中展示计划范围、总发生次数和完成进度。
4. 编辑时修改整条任务系列。

### 4.3 今日任务页

1. 按 `date === 今天` 筛选实例。
2. 只展示今天要处理的发生记录。
3. 组合任务在列表中展示为“大框 + 子任务小框”。
4. 支持删除本次发生、删除整个任务、到本次为止结束重复。

## 5. 行为统一原则

虽然任务总览页、项目进度页、今日任务页的布局不同，但以下行为必须一致：

1. 时间冲突校验一致。
2. 实例完成规则一致。
3. 删除单次发生与删除整条任务的边界一致。
4. 热度图和趋势图都按真实完成时间统计。
